# Правило: НЕТ автокоммитам

## ⛔ Запрет

**НИКОГДА** не выполняй команды `git commit` и `git push` без **ЯВНОЙ** команды пользователя.

## Когда НЕЛЬЗЯ коммить

❌ **Никогда не коммить автоматически:**
- После завершения задачи
- По запросу "обнови документацию"
- Когда пользователь сказал "измени файлы"
- В конце разговора
- После создания/редактирования файлов
- По запросу "задай изменения"
- Когда всё готово

## Когда МОЖНО коммить

✅ **Только при явной команде:**
- "закоммить изменения"
- "git commit -m '...'"
- "отправь в репозиторий"
- "git push"
- "сделай коммит"

## Что делать вместо автокоммита

1. **Сообщить о готовности:**
   ```
   ✅ Изменения готовы:
   - Файл X изменён
   - Файл Y создан
   - Тесты проходят
   
   Хочешь закоммить?
   ```

2. **Ждать явного подтверждения** перед выполнением git-команд

3. **Никогда не предполагай**, что пользователь хочет закоммить

## Исключения

**НЕТ ИСКЛЮЧЕНИЙ.** Это правило всегда действует.

## Нарушение правила

Если ты забыл и собрался закоммить без команды:
- ОСТАНОВИСЬ
- Спроси: "Хочешь закоммить изменения?"
- Жди ответа

---

**Приоритет: КРИТИЧЕСКИЙ** 🚨

Это правило имеет наивысший приоритет и не должно игнорироваться.
